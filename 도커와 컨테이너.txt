Docker , 도커 

-컨테이너 기반의 가상화 플랫폼
-컨테이너라는 단위로 데이터, 자원, 소프트웨어를 격리시켜주는 소프트웨어

 도커 -> 컨테이너 X 

*컨테이너
-소프트 웨어를 실행시키기 위해 생성된 격리공간
-특정 소프트웨어를 실행하기 위한 모든 환경이 포함되어 있다.

*컨테이너 , 가상머신 차이점
1) 컨테이너
- 운영체제 수준의 가상화

- 호스트의 커널을 공유한다.

- VM의 비해서 매우 가볍다.

-생성과 삭제가 매우 빠르다.

-격리 수준이 VM에 비해 떨어진다.

2) 가상 머신

-  하드웨어 수준의 가상화

- 많은 자원 소비

- 컨테이너와 비교해 격리 수준이 높다.

- 자원 할당량 설정 가능

*docker 관련 명령어 
run : 컨테이너 생성 + 실행
stop : 컨테이너 중지
start : 컨테이너 실행
rm : 컨테이너 삭제
ps : 컨테이너 확인

docker run --name itbank -d ubuntu/apache2
			(이미지)
도커 생성 명령어
docker start itbank
실행 명령어

docker ps
docker ps -a

*도커 이미지
- 도커 컨테이너를 생성하는데 사용되는 이미지

* 해당 이미지에 관련된 소프트웨어를 위해 필요한 모든 것이 포함되어 있다.

* 주로 Docker Hub와 같은 리포지토리에 저장되고 공유된다.

* 포트 포워딩 옵션
-p [포스트 포트]:[컨테이너 포트]

-호스트 포트로 접근 시 컨테이너 포트로 연결

-p 8081:80

컨테이너 내부접속
# docker exec -i -t [컨테이너 이름] /bin/bash

탈출 -> exit

*컨테이너 파일 복사
#docker cp [원본] [복사 경로]
컨테이너 경로 표현: [컨테이너 이름] : [컨테이너 상의 경로]
